# 🔄 自动重试机制说明

## ✨ 功能说明

平台现已支持**智能自动重试**：如果审核不通过，系统会自动重新生成作品，直到审核达标或达到最大重试次数。

---

## 🎯 核心特性

### 1️⃣ **智能重试循环**
- ✅ 审核不通过 → 自动重新生成
- ✅ 最多尝试5次
- ✅ 每次间隔3秒（避免API限流）
- ✅ 实时显示尝试进度

### 2️⃣ **详细日志记录**
- 📝 记录每次尝试的结果
- 📊 显示评分和审核状态
- 🔍 标注失败原因
- 💡 提供优化建议

### 3️⃣ **用户友好提示**
- 🔔 实时通知当前进度
- ⏳ 显示等待时间
- ✅ 成功时庆祝提示
- ⚠️ 失败时给出建议

### 4️⃣ **历史记录追踪**
- 📚 保存所有尝试的作品
- 🔄 标注尝试次数
- 📅 记录创建时间
- 🏷️ 显示审核状态

## ⚙️ 配置参数

### 最大重试次数
```javascript
const MAX_RETRIES = 5;  // 默认5次
```

**为什么是5次？**
- ✅ 足够多次尝试找到好作品
- ✅ 不会无限循环浪费资源
- ✅ 平衡质量和效率
- ✅ 给用户合理期待

**可以修改吗？**
可以！在代码中修改 `MAX_RETRIES` 的值：
```javascript
const MAX_RETRIES = 3;  // 改为3次
const MAX_RETRIES = 10; // 改为10次
```

### 重试间隔时间
```javascript
await this.delay(3000);  // 默认3秒
```

**为什么是3秒？**
- ✅ 避免API限流
- ✅ 给用户心理准备时间
- ✅ 不过长也不过短
- ✅ 平衡速度和体验

## 🔧 技术实现

### 核心逻辑

```javascript
async startFullWorkflow() {
    const MAX_RETRIES = 5;
    let attempt = 0;
    let auditPassed = false;
    
    while (attempt < MAX_RETRIES && !auditPassed) {
        attempt++;
        
        // 1. 生成歌词
        const lyrics = await this.lyricsAgent.generate(params);
        
        // 2. 生成编曲
        const arrangement = await this.arrangementAgent.generate(...);
        
        // 3. 评估质量
        const evaluation = await this.evaluationAgent.evaluate(...);
        
        // 4. 审核检查
        const audit = await this.auditAgent.audit(...);
        
        // 5. 检查结果
        if (audit.passed) {
            auditPassed = true;
            // 保存并展示
        } else {
            // 等待后重试
            await this.delay(3000);
        }
    }
}
```

### 错误处理

```javascript
try {
    // 创作流程
} catch (error) {
    console.error(`第${attempt}次尝试出错:`, error);
    
    if (attempt < MAX_RETRIES) {
        // 继续重试
        await this.delay(2000);
    }
}
```

## ❓ 常见问题

### Q: 为什么要自动重试？
A: 
- AI生成有随机性，多次尝试可提高质量
- 避免用户手动点击重试的麻烦
- 确保最终作品达到质量标准

### Q: 重试会消耗更多API额度吗？
A: 是的。每次尝试都会调用API。如果使用付费API，请注意成本。

### Q: 可以中途停止吗？
A: 当前版本不支持。未来会添加"取消"按钮。

### Q: 重试时参数会变吗？
A: 不会。使用相同的创作参数，但AI生成的内容会不同（随机性）。

### Q: 如果5次都不过怎么办？
A: 
1. 系统会保存最后一次作品供参考
2. 显示详细的失败原因和建议
3. 建议手动调整参数后重试

### Q: 可以在哪里看到尝试次数？
A: 
1. 通知提示中显示
2. 审核日志中有详细记录
3. 历史记录中标注尝试次数

### Q: 重试会影响性能吗？
A: 
- 每次尝试约15-20秒
- 5次最多约2分钟
- 后台运行，不影响其他操作

### Q: 可以自定义重试次数吗？
A: 当前固定为5次。未来版本会在设置中添加自定义选项。

### Q: 为什么有时很快通过，有时很慢？
A: 
- AI生成的随机性
- 审核标准的严格程度
- 创作参数的复杂度

### Q: 重试时会覆盖之前的作品吗？
A: 
- 不会覆盖
- 每次尝试都是独立的
- 只有最终通过的才会保存到历史
